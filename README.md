MongoDB Dashboard – Grazioso Salvare

In this course, I learned how important it is to write programs that are organized and easy to maintain. One of the biggest things that helped with this was the CRUD Python module I created in Project One. Instead of writing database code directly inside my dashboard in Project Two, I reused that module to handle all the database operations. That made my code cleaner and easier to manage. If I ever needed to change how the database connection worked or adjust a query, I could update it in one place instead of searching through the entire dashboard file.

Working this way made the project feel more structured and realistic. It showed me how separating responsibilities in code makes programs more adaptable. In the future, I could reuse that same CRUD module in other applications that connect to MongoDB, like a web app, an API, or another data dashboard.

When I approach a problem as a computer scientist, I try to break it down into smaller parts. For this project, I started by understanding what Grazioso Salvare needed. I looked at the requirements and figured out how the database, filters, table, and map would all work together. Instead of just focusing on writing code, I focused on how everything connects. This was different from some earlier assignments where I was mainly solving individual programming problems. Here, I had to think about system design, data flow, and user interaction at the same time.

Computer scientists build systems that help people use data in better ways. In this project, the dashboard helps an organization like Grazioso Salvare filter and visualize animal rescue data more efficiently. Instead of manually going through records, users can interact with the dashboard to quickly find the information they need. That saves time and helps with decision-making. Projects like this show how software can directly support real-world organizations and improve how they operate.
